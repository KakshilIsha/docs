# Steps

Steps are the building blocks of registration flows in the Global PRS system. Each step represents a specific action or data collection point in the registration process. This section covers the different types of steps, their configuration, and best practices.

## Step Types

### 1. Form Steps
- Data collection forms
- Health declarations
- Feedback forms
- Surveys

### 2. Payment Steps
- Fee collection
- Payment gateway integration
- Receipt generation
- Refund handling

### 3. Verification Steps
- Identity verification
- Document validation
- Email verification
- Phone verification

### 4. Integration Steps
- External system calls
- Data synchronization
- API integrations
- Webhook triggers

## Step Configuration

### 1. Basic Structure
```json
{
  "stepId": "personal_info",
  "type": "form",
  "title": "Personal Information",
  "description": "Please provide your basic details",
  "required": true,
  "order": 1,
  "config": {
    "formId": "personal_info_form",
    "validation": "strict",
    "savePartial": true
  }
}
```

## Step Implementation

### 1. Form Steps
```json
{
  "type": "form",
  "config": {
    "formId": "registration_form",
    "sections": ["personal", "contact"],
    "validation": "strict"
  }
}
```

### 2. Payment Steps
```json
{
  "type": "payment",
  "config": {
    "amount": "{program.fee}",
    "currency": "USD",
    "gateway": "stripe",
    "methods": ["card", "upi"]
  }
}
```

### 3. Verification Steps
```json
{
  "type": "verification",
  "config": {
    "method": "email",
    "expiry": 3600,
    "retries": 3
  }
}
```
